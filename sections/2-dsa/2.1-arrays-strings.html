2.1Arrays, Strings & Sliding Window

2.1.1 When to Use

Pattern
  • Contiguous subarray/substring problems
  • Fixed or variable size window
  • Maximum/minimum in a range

2.1.2 Template: Variable-Size Window

Pattern
// Find longest substring with at most K distinct characters - O(n)
public int longestSubstringKDistinct(String s, int k) {
    Map<Character, Integer> freq = new HashMap<>();
    int left = 0, maxLen = 0;

    for (int right = 0; right < s.length(); right++) {
        freq.merge(s.charAt(right), 1, Integer::sum);  // Expand window

        while (freq.size() > k) {                       // Shrink until valid
            char c = s.charAt(left);
            freq.merge(c, -1, Integer::sum);
            if (freq.get(c) == 0) freq.remove(c);
            left++;
        }
        maxLen = Math.max(maxLen, right - left + 1);    // Update answer
    }
    return maxLen;
}

2.1.3 Two Pointers Pattern

Pattern
// 3Sum: Find all triplets summing to zero - O(n^2)
public List<List<Integer>> threeSum(int[] nums) {
    Arrays.sort(nums);  // Sort enables two-pointer technique
    List<List<Integer>> result = new ArrayList<>();

    for (int i = 0; i < nums.length - 2; i++) {
        if (i > 0 && nums[i] == nums[i-1]) continue; // Skip duplicates

        int left = i + 1, right = nums.length - 1;
        while (left < right) {
            int sum = nums[i] + nums[left] + nums[right];
            if (sum == 0) {
                result.add(List.of(nums[i], nums[left], nums[right]));
                while (left < right && nums[left] == nums[left+1]) left++;
                while (left < right && nums[right] == nums[right-1]) right--;
                left++; right--;
            } else if (sum < 0) left++;
            else right--;
        }
    }
    return result;
}
// Trade-off: Sorting O(n log n) enables O(n) two-pointer scan
// Alternative: HashSet approach is O(n^2) time, O(n) space but harder to deduplicate

2.1.4 Prefix Sum Pattern

Pattern
// Subarray sum equals K - O(n) with prefix sum + hashmap
public int subarraySum(int[] nums, int k) {
    Map<Integer, Integer> prefixCount = new HashMap<>();
    prefixCount.put(0, 1);  // Empty prefix
    int sum = 0, count = 0;

    for (int num : nums) {
        sum += num;
        count += prefixCount.getOrDefault(sum - k, 0); // How many prefixes make valid subarray
        prefixCount.merge(sum, 1, Integer::sum);
    }
    return count;
}
// Key insight: if prefix[j] - prefix[i] = k, then subarray [i+1..j] sums to k

2.1.5 Binary Search Patterns

Pattern
// O(log n). The hard part is loop invariants and overflow, not the idea.
// Always use mid = lo + (hi - lo) / 2 to avoid integer overflow.

// 1) Classic: find exact target
int search(int[] a, int target) {
    int lo = 0, hi = a.length - 1;          // closed interval [lo, hi]
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        if (a[mid] == target) return mid;
        else if (a[mid] < target) lo = mid + 1;
        else hi = mid - 1;
    }
    return -1;
}

// 2) Leftmost insertion point (lower_bound) - first index where a[i] >= target
int lowerBound(int[] a, int target) {
    int lo = 0, hi = a.length;              // half-open [lo, hi)
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        if (a[mid] < target) lo = mid + 1;
        else hi = mid;
    }
    return lo;                              // in [0, a.length]
}

// 3) Search in rotated sorted array - one half is always sorted
int searchRotated(int[] a, int target) {
    int lo = 0, hi = a.length - 1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        if (a[mid] == target) return mid;
        if (a[lo] <= a[mid]) {              // left half sorted
            if (a[lo] <= target && target < a[mid]) hi = mid - 1; else lo = mid + 1;
        } else {                            // right half sorted
            if (a[mid] < target && target <= a[hi]) lo = mid + 1; else hi = mid - 1;
        }
    }
    return -1;
}

Binary Search on the Answer

The most missed senior pattern: when the answer is a number in a monotonic range (min capacity to ship in D days, smallest divisor, Koko eating bananas), binary-search the answer space and use a feasibility check canDo(x) as the predicate. Complexity becomes O(n · log(range)).

Interview Tip

Pick one template and stick to it to avoid off-by-one bugs. Verbalize the invariant ("everything left of lo is < target"). If you see "sorted" or "minimize the maximum / maximize the minimum", binary search should be your first thought.
