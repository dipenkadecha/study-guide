2.5Dynamic Programming & Greedy

2.5.1 DP Problem-Solving Framework

Pattern
  1. Define state: What information do I need to make a decision at each step?
  2. Define transition: How does current state relate to smaller subproblems?
  3. Base case: What's the answer for the smallest subproblem?
  4. Optimize space: Do I need the full table or just previous row?

2.5.2 Pattern: Knapsack (0/1)

Pattern
// 0/1 Knapsack - O(n*W) time, O(W) space (optimized)
public int knapsack(int[] weights, int[] values, int capacity) {
    int n = weights.length;
    int[] dp = new int[capacity + 1];

    for (int i = 0; i < n; i++) {
        // Traverse RIGHT to LEFT to avoid using same item twice
        for (int w = capacity; w >= weights[i]; w--) {
            dp[w] = Math.max(dp[w], dp[w - weights[i]] + values[i]);
        }
    }
    return dp[capacity];
}
// Variant: Unbounded knapsack -> traverse LEFT to RIGHT (reuse items)
// Variant: Count ways -> dp[w] += dp[w - weights[i]]

2.5.3 Pattern: Longest Common Subsequence

Pattern
// LCS - O(m*n) time, O(min(m,n)) space
// Applications: diff algorithms, DNA sequence alignment, version control
public int longestCommonSubsequence(String text1, String text2) {
    if (text1.length() < text2.length()) return longestCommonSubsequence(text2, text1);
    int[] dp = new int[text2.length() + 1];

    for (int i = 1; i <= text1.length(); i++) {
        int prev = 0;
        for (int j = 1; j <= text2.length(); j++) {
            int temp = dp[j];
            if (text1.charAt(i-1) == text2.charAt(j-1))
                dp[j] = prev + 1;
            else
                dp[j] = Math.max(dp[j], dp[j-1]);
            prev = temp;
        }
    }
    return dp[text2.length()];
}

2.5.4 Greedy: When to Use

Pattern
// Greedy works when: local optimal choice leads to global optimum
// Proof technique: exchange argument or greedy stays ahead

// Interval Scheduling - Maximum non-overlapping intervals
// Sort by end time, always pick earliest ending
public int maxNonOverlapping(int[][] intervals) {
    Arrays.sort(intervals, (a, b) -> a[1] - b[1]);  // Sort by end time
    int count = 1, end = intervals[0][1];
    for (int i = 1; i < intervals.length; i++) {
        if (intervals[i][0] >= end) {  // Non-overlapping
            count++;
            end = intervals[i][1];
        }
    }
    return count;
}

Trade-off: DP vs Greedy

AspectDPGreedy
Optimal substructureRequiredRequired
Overlapping subproblemsRequiredNot needed
Greedy choice propertyNot neededRequired (must prove)
ComplexityHigher (explores all)Lower (one choice per step)
CorrectnessAlways correctMust prove greedy works

2.5.5 Backtracking

Pattern
// Backtracking = DFS over a decision tree: choose -> explore -> un-choose.
// Use cases: subsets, permutations, combinations, N-Queens, word search, Sudoku.

// 1) Subsets - O(2^n) results, each built incrementally
void subsets(int[] nums, int start, List<Integer> path, List<List<Integer>> out) {
    out.add(new ArrayList<>(path));          // every node is a valid subset
    for (int i = start; i < nums.length; i++) {
        path.add(nums[i]);                    // choose
        subsets(nums, i + 1, path, out);      // explore (i+1 avoids reuse)
        path.remove(path.size() - 1);         // un-choose (backtrack)
    }
}

// 2) Permutations - O(n!) using a 'used' marker
void permute(int[] nums, boolean[] used, List<Integer> path, List<List<Integer>> out) {
    if (path.size() == nums.length) { out.add(new ArrayList<>(path)); return; }
    for (int i = 0; i < nums.length; i++) {
        if (used[i]) continue;
        used[i] = true; path.add(nums[i]);
        permute(nums, used, path, out);
        used[i] = false; path.remove(path.size() - 1);
    }
}

// 3) Pruning is what makes it tractable: in N-Queens / Sudoku, check validity
//    BEFORE recursing so whole branches are skipped (the "bound" in branch-and-bound).

Trade-off: Backtracking vs DP

Backtracking enumerates solutions (you need the actual subsets/arrangements) and is exponential. DP counts or optimizes by reusing overlapping subproblems. If the question asks "how many ways" or "max value", prefer DP; if it asks "list all valid configurations", it's backtracking. To deduplicate (e.g. subsets with duplicates), sort first and skip i > start && nums[i] == nums[i-1].

Key Tip

State the template out loud: choose → explore → un-choose, plus "where can I prune?". Always give the complexity in terms of the solution space (2^n subsets, n! permutations) and note the O(n) path-copy cost on each leaf.
