2.4Heaps, Priority Queues & Tries

2.4.1 Top-K Pattern

Pattern
// Find K largest elements - O(n log k) using min-heap of size K
public int[] topKFrequent(int[] nums, int k) {
    Map<Integer, Integer> freq = new HashMap<>();
    for (int n : nums) freq.merge(n, 1, Integer::sum);

    // Min-heap by frequency - keeps K largest
    PriorityQueue<Integer> pq = new PriorityQueue<>(
        Comparator.comparingInt(freq::get));

    for (int num : freq.keySet()) {
        pq.offer(num);
        if (pq.size() > k) pq.poll();  // Remove smallest frequency
    }
    return pq.stream().mapToInt(Integer::intValue).toArray();
}
// Trade-off: Heap O(n log k) vs QuickSelect O(n) average
// Heap: guaranteed O(n log k), streaming-friendly
// QuickSelect: O(n) average but O(n^2) worst case, needs all data upfront

2.4.2 Merge K Sorted Streams

Pattern
// Merge K sorted lists - O(N log K) where N = total elements
// Production use: merging sorted SSTable files in LSM-tree compaction
public ListNode mergeKLists(ListNode[] lists) {
    PriorityQueue<ListNode> pq = new PriorityQueue<>(
        Comparator.comparingInt(a -> a.val));

    for (ListNode head : lists)
        if (head != null) pq.offer(head);

    ListNode dummy = new ListNode(0), tail = dummy;
    while (!pq.isEmpty()) {
        ListNode min = pq.poll();
        tail.next = min;
        tail = tail.next;
        if (min.next != null) pq.offer(min.next);
    }
    return dummy.next;
}

2.4.3 Trie (Prefix Tree)

Pattern
// Trie - O(L) operations where L = word length
// Use cases: autocomplete, spell check, IP routing, word search
class Trie {
    private TrieNode root = new TrieNode();

    static class TrieNode {
        TrieNode[] children = new TrieNode[26];
        boolean isEnd;
        int count; // For ranking in autocomplete
    }

    public void insert(String word) {
        TrieNode node = root;
        for (char c : word.toCharArray()) {
            int idx = c - 'a';
            if (node.children[idx] == null)
                node.children[idx] = new TrieNode();
            node = node.children[idx];
        }
        node.isEnd = true;
        node.count++;
    }

    // Autocomplete: find all words with given prefix
    public List<String> autocomplete(String prefix, int limit) {
        TrieNode node = root;
        for (char c : prefix.toCharArray()) {
            node = node.children[c - 'a'];
            if (node == null) return Collections.emptyList();
        }
        List<String> results = new ArrayList<>();
        dfs(node, new StringBuilder(prefix), results, limit);
        return results;
    }

    // Collect words under `node` in lexicographic order until we hit `limit`
    private void dfs(TrieNode node, StringBuilder path,
                     List<String> results, int limit) {
        if (results.size() >= limit) return;
        if (node.isEnd) results.add(path.toString());
        for (char c = 'a'; c <= 'z'; c++) {
            TrieNode child = node.children[c - 'a'];
            if (child != null) {
                path.append(c);
                dfs(child, path, results, limit);
                path.deleteCharAt(path.length() - 1); // backtrack
            }
        }
    }
}

Interview Tip

Trie vs HashMap for prefix search: HashMap needs O(N*L) to check all keys for prefix. Trie finds all prefix matches in O(P + M) where P=prefix length, M=matches. For autocomplete with millions of words, Trie wins.
