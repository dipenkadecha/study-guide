2.2Hashing & HashMaps

2.2.1 HashMap Internal Structure (Java 8+)

Core
  • Array of Node[] buckets (default 16, load factor 0.75)
  • Bucket collision: linked list -> when bucket has 8+ nodes, converts to Red-Black Tree (O(n) -> O(log n))
  • Resize: double capacity, rehash all entries (expensive!)
// Pre-size HashMap to avoid rehashing - production optimization
// Expected 10000 entries, load factor 0.75 -> need capacity 13334
Map<String, Order> orderCache = new HashMap<>((int)(10000 / 0.75) + 1);

// Compute patterns - atomic read-modify-write
cache.computeIfAbsent(key, k -> expensiveComputation(k));  // Only compute if missing
cache.merge(key, 1, Integer::sum);                          // Atomic increment

2.2.2 ConcurrentHashMap - Thread-Safe Without Global Lock

Core
// ConcurrentHashMap: lock striping (Java 8+: CAS + synchronized on node)
// No null keys or values (unlike HashMap)
ConcurrentHashMap<String, AtomicLong> metrics = new ConcurrentHashMap<>();

// WRONG - race condition (check-then-act)
if (!metrics.containsKey(key)) { metrics.put(key, new AtomicLong(0)); }
metrics.get(key).incrementAndGet();

// RIGHT - atomic compound operation
metrics.computeIfAbsent(key, k -> new AtomicLong(0)).incrementAndGet();

// Bulk parallel operations (Java 8+)
long total = metrics.reduceValuesToLong(
    1000,               // parallelism threshold: go parallel if size > 1000
    AtomicLong::get,    // transformer
    0L,                 // identity
    Long::sum           // reducer
);

2.2.3 Consistent Hashing (Distributed Systems)

Core
// Consistent hashing: minimize key redistribution when nodes change
// Without: adding 1 node to 10 redistributes ~90% keys (modulo changes)
// With: adding 1 node redistributes only ~10% keys (K/N average)
public class ConsistentHash<T> {
    private final TreeMap<Long, T> ring = new TreeMap<>();
    private final int virtualNodes;  // 100-200 vnodes per physical node

    public void addNode(T node) {
        for (int i = 0; i < virtualNodes; i++) {
            long hash = hash(node.toString() + "#" + i);
            ring.put(hash, node);
        }
    }

    public T getNode(String key) {
        long hash = hash(key);
        Map.Entry<Long, T> entry = ring.ceilingEntry(hash);
        return (entry != null) ? entry.getValue() : ring.firstEntry().getValue();
    }
}
// Trade-off: More virtual nodes = better distribution but more memory
// Production: 150-200 vnodes per node is sweet spot
