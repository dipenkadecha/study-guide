2.3Trees & Graphs

2.3.1 BFS Level-Order (Queue-based)

Pattern
// BFS - Level order traversal - O(n) time, O(w) space where w = max width
public List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> result = new ArrayList<>();
    if (root == null) return result;
    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);

    while (!queue.isEmpty()) {
        int levelSize = queue.size();  // Snapshot current level size
        List<Integer> level = new ArrayList<>();
        for (int i = 0; i < levelSize; i++) {
            TreeNode node = queue.poll();
            level.add(node.val);
            if (node.left != null) queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
        result.add(level);
    }
    return result;
}

2.3.2 DFS Patterns

Pattern
// Iterative DFS with explicit stack - avoids StackOverflow for deep (100K+) trees.
// (The recursive one-liner is cleaner but blows the call stack on skewed trees.)
public int maxDepth(TreeNode root) {
    if (root == null) return 0;
    Deque<TreeNode> nodes = new ArrayDeque<>();
    Deque<Integer> depths = new ArrayDeque<>();
    nodes.push(root);
    depths.push(1);
    int max = 0;
    while (!nodes.isEmpty()) {
        TreeNode node = nodes.pop();
        int depth = depths.pop();
        max = Math.max(max, depth);
        if (node.left != null)  { nodes.push(node.left);  depths.push(depth + 1); }
        if (node.right != null) { nodes.push(node.right); depths.push(depth + 1); }
    }
    return max;
}

// Lowest Common Ancestor - O(n) time, O(h) space
public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
    if (root == null || root == p || root == q) return root;
    TreeNode left = lowestCommonAncestor(root.left, p, q);
    TreeNode right = lowestCommonAncestor(root.right, p, q);
    if (left != null && right != null) return root;  // Found on both sides
    return left != null ? left : right;
}

2.3.3 Topological Sort (Kahn's BFS)

Advanced
// Use case: build systems, task scheduling, course prerequisites
// Only works on DAG (Directed Acyclic Graph)
public List<Integer> topologicalSort(int n, int[][] edges) {
    List<List<Integer>> adj = new ArrayList<>();
    int[] inDegree = new int[n];
    for (int i = 0; i < n; i++) adj.add(new ArrayList<>());

    for (int[] e : edges) {
        adj.get(e[0]).add(e[1]);
        inDegree[e[1]]++;
    }

    Queue<Integer> queue = new LinkedList<>();
    for (int i = 0; i < n; i++) if (inDegree[i] == 0) queue.offer(i);

    List<Integer> order = new ArrayList<>();
    while (!queue.isEmpty()) {
        int node = queue.poll();
        order.add(node);
        for (int neighbor : adj.get(node)) {
            if (--inDegree[neighbor] == 0) queue.offer(neighbor);
        }
    }
    return order.size() == n ? order : Collections.emptyList(); // Cycle detection
}

2.3.4 Dijkstra's Shortest Path

Advanced
// Weighted graph shortest path - O((V+E) log V) with priority queue
// Use case: routing, network latency, cost optimization
public int[] dijkstra(int n, List<int[]>[] adj, int src) {
    int[] dist = new int[n];
    Arrays.fill(dist, Integer.MAX_VALUE);
    dist[src] = 0;

    PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> a[1] - b[1]);
    pq.offer(new int[]{src, 0});

    while (!pq.isEmpty()) {
        int[] curr = pq.poll();
        int u = curr[0], d = curr[1];
        if (d > dist[u]) continue;  // Skip outdated entry

        for (int[] edge : adj[u]) {
            int v = edge[0], w = edge[1];
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                pq.offer(new int[]{v, dist[v]});
            }
        }
    }
    return dist;
}
// Trade-off: Dijkstra O((V+E)logV) vs Bellman-Ford O(VE)
// Dijkstra: faster, no negative edges
// Bellman-Ford: handles negative edges, detects negative cycles

Trade-off: B-Tree vs LSM-Tree

  • B-Tree (PostgreSQL, MySQL): Read-optimized, in-place updates, good for read-heavy OLTP. Write amplification from random I/O.
  • LSM-Tree (Cassandra, RocksDB): Write-optimized, sequential writes to memtable then SSTables. Read amplification from multiple levels. Compaction overhead.
  • Decision: Read:Write ratio > 80:20 -> B-Tree. Write-heavy (logs, time-series, messaging) -> LSM-Tree

2.3.6 Union-Find (Disjoint Set Union)

Pattern
// Near O(1) amortized (inverse Ackermann) per op with both optimizations.
// Use cases: connected components, cycle detection in undirected graphs,
// Kruskal's MST, account merging, "number of provinces / islands" (offline).
class UnionFind {
    private final int[] parent, rank;
    private int components;

    UnionFind(int n) {
        parent = new int[n];
        rank = new int[n];
        components = n;
        for (int i = 0; i < n; i++) parent[i] = i;
    }

    int find(int x) {                       // path compression
        while (parent[x] != x) {
            parent[x] = parent[parent[x]];  // halve the path
            x = parent[x];
        }
        return x;
    }

    boolean union(int a, int b) {           // union by rank
        int ra = find(a), rb = find(b);
        if (ra == rb) return false;         // already connected -> cycle edge
        if (rank[ra] < rank[rb]) { int t = ra; ra = rb; rb = t; }
        parent[rb] = ra;
        if (rank[ra] == rank[rb]) rank[ra]++;
        components--;
        return true;
    }

    int count() { return components; }      // # of disjoint sets
}

// Cycle detection: union() returns false when both ends share a root.
// Counting components: start with n, each successful union shrinks by one.

Trade-off: Union-Find vs BFS/DFS

For dynamic connectivity (edges arriving over time, repeated "are these connected?" queries) Union-Find is far better — each query is ~O(1) amortized. BFS/DFS recomputes components in O(V+E) per query. But Union-Find only answers connectivity; it can't give the actual path, and it doesn't handle edge deletion.

Key Tip

Reach for DSU when you hear "merge groups", "connected components", "redundant connection", or "MST". Always mention both optimizations (path compression + union by rank) — without them it degrades to O(n) per op.
