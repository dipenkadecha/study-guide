// Integration test with Testcontainers (real DB, not mocks)
@SpringBootTest
@Testcontainers
class OrderServiceIntegrationTest {

    @Container
    static PostgreSQLContainer<?> postgres = new PostgreSQLContainer<>("postgres:15")
        .withDatabaseName("orders_test");

    @Container
    static KafkaContainer kafka = new KafkaContainer(DockerImageName.parse("confluentinc/cp-kafka:7.4.0"));

    @DynamicPropertySource
    static void configure(DynamicPropertyRegistry registry) {
        registry.add("spring.datasource.url", postgres::getJdbcUrl);
        registry.add("spring.kafka.bootstrap-servers", kafka::getBootstrapServers);
    }

    @Test
    void shouldCreateOrderAndPublishEvent() {
        // Given
        OrderRequest request = new OrderRequest("cust-1", List.of(new LineItem("SKU-1", 2)));

        // When
        Order order = orderService.createOrder(request);

        // Then
        assertThat(order.status()).isEqualTo(OrderStatus.CREATED);
        assertThat(orderRepository.findById(order.id())).isPresent();

        // Verify Kafka event published
        ConsumerRecord<String, String> record = KafkaTestUtils.getSingleRecord(consumer, "orders");
        assertThat(record.value()).contains(order.id());
    }
}

// Contract test - ensure API compatibility
@SpringBootTest
@AutoConfigureStubRunner(ids = "com.myapp:inventory-service:+:stubs:8090")
class OrderServiceContractTest {
    @Test
    void shouldCallInventoryServiceCorrectly() {
        // Stub runner provides WireMock stub from inventory-service's contract
        StockResponse response = inventoryClient.checkStock("SKU-1");
        assertThat(response.available()).isTrue();
    }
}

Trade-off: Mocking vs Real Dependencies

  • Mocks (Mockito): Fast, isolated, but can drift from real behavior. Use for unit tests.
  • Testcontainers: Real DB/Kafka/Redis in Docker. Slower but catches integration issues. Use for integration tests.
  • Contract Tests: Verify API compatibility without running other services. Catches breaking changes at build time.
  • Rule of thumb: Mock what you don't own (external APIs). Use real instances for what you do own (your DB, your Kafka).
